Implementation Approaches

How we integrate the task-focused officer dashboard

A

Router-Driven Redirection

Approach: Modify appRouter to redirect to /officer/dashboard if user.role == 'officer'.

  • Cleanest code separation
  • Different navigation labels
  • Requires auth refresh on login
B

Dashboard Switcher

Approach: DashboardScreen becomes a wrapper that shows either CitizenView or OfficerView.

  • Easiest to implement
  • Maintains same URL structure
  • Navigation bar stays the same (User sees citizen tabs)

I recommend Option A (Router-Driven) because officers need a different mental model and navigation set (e.g., "Assigned", "Resolved" instead of "Map", "Chat").
